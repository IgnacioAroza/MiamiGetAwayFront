## 1. PUT → PATCH en reservationService.update

`reservationService.update` usaba `api.put` pero el backend pasó a requerir PUT solo para reemplazos completos y PATCH para actualizaciones parciales. Cambiado a `api.patch`.

Eliminados como código muerto (nunca se usaban):
- Thunk `updateReservation` en `reservationSlice.js`
- Handler `handleUpdateReservation` en `useReservation.js`

`ReservationManagement.jsx` siempre llamó `reservationService.update()` directamente (bypass Redux), por lo que el thunk nunca tuvo consumidores reales.

## 2. Error handling — root cause y fix

**Root cause:** Los servicios lanzan **strings** (`throw error.response?.data?.error || ... || 'fallback'`), no Error objects. Los thunks y componentes accedían a `error.message` que es `undefined` en un string → siempre mostraban el fallback genérico.

### Patrón correcto para thunks:

```js
const extractMsg = (error, fallback) =>
    typeof error === 'string' ? error : (error?.response?.data?.error || error?.response?.data?.message || error?.message || fallback);
// Usar en rejectWithValue:
return rejectWithValue(extractMsg(error, 'Error fallback'));
```

### Patrón correcto para catch en componentes:

```js
} catch (error) {
    const msg = typeof error === 'string' ? error : (error?.message || 'fallback');
    showError(msg);
}
```

### Bugs específicos encontrados y corregidos:

| Archivo | Bug | Síntoma |
|---------|-----|---------|
| `reservationSlice.js` | Thunks usaban `error.response?.data?.error` sobre string → undefined | Rate limit mostraba "Error fetching reservation" |
| `reservationPaymentSlice.js` | `rejectWithValue(error.message)` sobre string → `rejectWithValue(undefined)` | Errores swallowados, componentes crasheaban |
| `serviceSlice.js` | `handleApiError` hacía `throw` en vez de `return` → `rejectWithValue` nunca se ejecutaba → reducer crasheaba al destructurar `action.payload` (undefined) | Cars/Yachts/Villas: skeleton infinito |
| `userSlice.js` | `rejectWithValue(error.response?.data)` pasaba el objeto completo | React crash "Objects are not valid as React child" → blank en Clients |
| `summarySlice.js` | `rejectWithValue(error.message)` sobre string → `undefined` | `state.error = undefined` → blank en Home |
| `PaymentSection.jsx` | `catch (error)` pisaba la función `error()` de useToast | Errores de pago swallowados silenciosamente (TypeError: error is not a function) |
| `ReservationManagement.jsx` | `error.message \|\| 'fallback'` sobre string | Siempre mostraba fallback |
| `PaymentsForm.jsx`, `PaymentsList.jsx`, `AdminLogin.jsx` | Mismo patrón | Ídem |

### Variable shadowing en PaymentSection (bug silencioso):
```js
const { error } = useToast(); // error = función
} catch (error) {             // pisa la función
    error('mensaje');         // llamaba al string como función → TypeError silencioso
```
